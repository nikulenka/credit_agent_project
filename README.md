# 🏦 credit_agent_project

Система автоматизированного принятия решений по кредитованию на базе LangGraph и LLM (OpenAI).

Описание

Этот проект реализует агентную pipeline для оценки кредитных заявок: извлечение финансовых метрик из документов, вычисление ключевых показателей и принятие решения с использованием нескольких стратегий (simple, debate, tools) и проверок (guardrails, policies).

Ключевые файлы

- `agent.py` — определение StateGraph (LangGraph), ноды рассуждений, guardrails, политики и сбор результата в модель `CreditDecision`.
- `main.py` — пример запуска: сканирование папки `data/`, вычисление метрик, запуск графа и вывод финального решения.
- `scanner.py` — функции для парсинга финансовых документов и подготовки данных (используется `pdfplumber` и другие утилиты).
- `models.py` — Pydantic-модели (`CreditApplication`, `ProcessedMetrics`, `CreditDecision`) для строгой типизации данных.
- `requirements.txt` — зависимости проекта (langchain, langgraph, pydantic, python-dotenv и т.д.).

Требования

- Python 3.8+
- Установите зависимости:

```bash
pip install -r requirements.txt
```

- Установите OPENAI_API_KEY в окружении или в файле `.env` в корне репозитория.

Запуск

1. Подготовьте папку `data/` с исходными финансовыми документами (PDF, CSV и т.д.).
2. Создайте `.env` с переменной `OPENAI_API_KEY` (и при необходимости другими переменными).
3. Запустите:

```bash
python main.py
```

Архитектура

Пайплайн состоит из следующих шагов:

1. Сканирование документов и извлечение структурированных данных (`scanner.parse_financial_documents`).
2. Вычисление метрик (`calculate_metrics`).
3. Инициация `StateGraph` (LangGraph) — `build_credit_graph()`:
   - `supervisor` — выбирает стратегию (simple/debate/tools)
   - reasoning nodes — различные стратегии вывода
   - `guardrails` — проверка целостности и минимального уровня уверенности
   - `policies` — применение банковских политик
   - `execution` — формирование `CreditDecision`

Замечания по безопасности

- Не храните API-ключи в репозитории.
- Ограничьте размер и формат загружаемых документов.
- Внимательно проверяйте вывод LLM перед использованием в production.

Тесты и разработка

- Для разработки можно добавлять тестовые документы в `data/` и менять параметры в `main.py`.
